# Tenji Roadmap

## Phase 1: Foundation

- finalize token and airdrop contracts
- complete deployment flow
- publish GitHub docs
- establish visual identity and mascot direction

## Phase 2: Public Launch Surface

- launch the public website
- publish refined tokenomics and a fuller litepaper
- publish official deployed addresses
- verify contracts on the target explorer

## Phase 3: Distribution

- run the initial airdrop campaign
- onboard early community members
- publish claim instructions and user guides
- monitor operational behavior and feedback

## Phase 4: Community Identity

- expand lore, branding, and content around Tenji
- build community-native distribution and campaign ideas
- improve public analytics and information surfaces

## Phase 5: Ecosystem Direction

- evaluate additional community tooling
- evaluate listing readiness and liquidity strategy
- expand documentation based on real usage

## Roadmap Philosophy

Tenji should grow in layers:

- first, the contract core must be understandable
- second, the public narrative must be coherent
- third, distribution should be executed cleanly
- only then should broader ecosystem ambitions be explored

The roadmap is directional, not a promise of fixed delivery dates.
